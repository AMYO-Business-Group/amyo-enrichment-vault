# Promotion Protocol

## Stages

1. Raw note in vault.
2. Structured analysis in vault.
3. Record or sync to Notion Enrichment Hub.
4. Approval gate.
5. Promotion package.
6. Main repo issue, ADR, or implementation plan.
7. Review package.
8. Validation and Notion sync.

## Approval gates

- `APPROVE SLICE PLANNING`
- `APPROVE IMPLEMENTATION PLAN`
- `APPROVE REPO CHANGES`

## Promotion package checklist

- [ ] Problem statement
- [ ] Workstream
- [ ] User persona
- [ ] Workflow trigger
- [ ] Desired outcome
- [ ] Scope
- [ ] Non-goals
- [ ] Existing table fit
- [ ] Data impact
- [ ] RLS impact
- [ ] UI routes
- [ ] Test plan
- [ ] Reversal plan
- [ ] Notion records linked or identified
- [ ] Decision owner
